Справка по CMS

Введение

Администрирование в данной CMS возможно на трёх уровнях с присвоением соответствующих статусов:

  • "globaladmin"
  • "admin"
  • "bloger"

После входа в CMS администратор попадает на главную страницу CMS (sit/cms/admin.php).

Главная

Права "globaladmin" позволяют:

  • видеть свой статус и имя пользователя.
  • видеть всех администраторов.
  • удалять, добавлять, изменять пользователей.
Права "admin" и "bloger" позволяют:
  • видеть свой статус и имя пользователя.

С главной страницы CMS можно перейти на главную страницу сайта или на страницу "О системе".

Управление файлами

Управление файлами возможно только с правами "globaladmin".

Доступны все операции, которые указаны на кнопках Горячие клавиши, создание изображений, смена дисков или всего пути. Можно открывать файлы в редакторе текста, изображения, в браузере или в режиме просмотра. Также система может определить по базе данных, как открывать файл.

Тип текст

На странице появляется отступ, увеличивается шрифт, добавляются некоторые возможности: переходить в редактор текста (по нажатию клавиши E), увеличивать или уменьшать масштаб текста.

Шаблоны

В CMS имеется поддержка шаблонов.

Пользователь, имеющий права globaladmin, может редактировать шаблоны, добавлять новые, удалять текущие, устанавливать шаблон/пакет шаблонов, анинсталировать шаблон/пакет шаблонов, удалять инсталятор шаблона/пакета шаблонов, запаковывать шаблон/пакет шаблонов.

Шаблон исаользует стили шаблонов и части////????.

К каждому шаблону приписан определенный стиль и таблица стилей: стили задают внешний вид страницы, на которой применён данный шаблон, а таблица стилей задает параметры, которые можно редактировать.

Конструкции шаблонов

  • <!--SCRIPT--> подключение скрипта JavaScript
  • <!--PHP--> подключение скрипта PHP
  • <!--STYLES--> подключение общих стилей CSS
  • <!--STYLE--> подключение стиля шаблона.

Две первые ///конструкции //// можно указывать в любом месте шаблона.

Двое последних необходимо указывать в области <head> </head>.

В остальном шаблоны соответствуют разметке HTML.

Части

Части можно устанавливать, удалять, изменять, добавлять, анинсталировать, запаковывать и удалять пакет.

Работа с частями

В CMS имеется поддержка частей шаблонов. При установке шаблона можно установить части, прилагаемые к нему.

Части предсвтавляют собой текст, который находится в файле, вставляемый на странице в определенном месте. Его можно редактировать отдельно от всего шаблона.

В шаблоне можно обозначать части так: <!--THE_название части-->. /////На этом месте будет вставлен текст части/////

Простые операции с файлами

Подгрузка файлов

Администратору, имеющему права globaladmin, разрешается подгружать файлы плагина, шаблоны/пакеты шаблонов, пакеты частей или другие файлы.

При наличии прав globaladmin можно получать файлтекст подгружаемого файла.

Файлтекст

Файлтекст - текст в сжатой шестнадцатиричной форме, который представляет бинарное содержимое файла.////

Его можно использовать для создания плагинов или хранения в текстовых файлах. ////

Скачивание файлов

Администратор, имеющий права globaladmin, может скачивать те файлы, которые можно подгружать (файлы скачиваются под именем download.php).

Страницы

Все страницы сайта (кроме служебных, например, 404.php или 405.php)можно создавать, удалять, изменять, добавлять. Администратор, имеющий права globaladmin, может обращаться к любой директории (даже к cms/). Служебные страницы тоже можно модифицировать, но это крайне нежелательно

Администратору, имеющему права admin недоступно написание в имени директории символа "." и набора символов "cms".

Администраторы с правами bloger имеют доступ только к файлам и папкам, находящимся в их личной директории. Путь до личеой папки в этом случае прописывается так: bloger/имя блогера. При этом, путь до создаваемой страницы не должен содержать символ ".".

О системе

На странице "О системе" можно узнать краткую информацию о CMS. Администратор с правами globaladmin имеет возможность проверить наличие обновлений, и, если они имеются, предлагается их установка. На этой же странице можно изменять путь, по которому CMS обращается к базе данных обновлений.

Обновления

Обновления бывают двух типов: update и new.

  • Тип update: CMS выполняет обновление отдельной части CMS.
  • Тип new: CMS "перезаливается" полностью (запускается инсталятор новой версии в режиме обновления).

Совет: путь, по которому CMS обращается к базе данных обновлений (параметр: путь обновлений), не следует менять без повода, так как это может оказаться путь, который ведёт на поддельную базу обновлений, вследствие чего злоумышленники могут "взломать" весь сервер. Отсутствует функция подгрузки файла через URL, так как это может понизить безопасность

Инстолятор CMS

Для установки CMS выполните следующие действия:

  1. Любыми средствами подгрызите файл инстолятора (install.php) в корневую папку сайта.
  2. Измените третью строку файла инстолятора на пароль.
  3. Перейдите на страницу инстолятора (имя сайта/install.php).

Винстоляторе имеется масса опций, которывые поясняются во время инсталяции.

Примеры опций:

  • Выбор устанавливаемых элементов.
  • Выбор режима обновления
  • Прочтение данного текста
