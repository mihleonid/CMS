Введение и краткое описание

Проект CMS Leonid является разработка Михайлова Леонида и представляет собой систему управления сайтом(Content Manage System). Данная система позволяет создавать сайты и управлять ими.

Однозначный и понятный интерфейс CMS Leonid организован при помощи HTML, JavaScript, CSS. Скрипты на стороне сервера созданы с помощью языка программирования PHP. Система отличается быстродействием, так как не использует внешних библиотек и готового кода.

В системе CMS Leonid возможно администрирование сайта несколькими пользователями, статусами которых возможно управлять.

Система имеет автоматическое обновление основного блока и полу-автоматическое (загрузка файла) плагинов, с помощью которых возможно расширять возможности CMS. Плагины (как и прочие модули и шаблоны) легко получить из встроенного магазина CMS (содержащего многочисленные разработки автора).

Интерфейс пользователя может быть настроен в соответствии с его пожеланиями и предпочтениями. Возможно выбирать тему для системы и фон сайта. Темы для удобства объединены в пакеты.

Для создания и редактирования страниц можно использовать удобный графический редактор (разработанный автором).

При операциях со страницами используются шаблоны. Шаблоны позволяют придать сайту единый вид. В шаблоне могут использоваться редко изменяемые параметры сайта (например, его название). Чтобы их редактировать без изменения всего шаблона имеется опция редактирования части. Возможно производить различные операции

Часть шаблона – это элемент такового, вынесенный в отдельный файл, доступ к которому предоставляет штатный интерфейс. Части объединены в пакеты для удобства использования.

Шаблоны создаются с помощью языков программирования HTML, CSS, JavaScript. Пользователи, не знакомые с ними, тоже могут редактировать шаблон, используя функцию изменения стилей шаблона.

Скриптовые модули - это модули PHP, CSS, JavaScript, которые подключаются на страницу. Например, часы.

Плагин - универсальный модуль системы. Они могут производить любые действия.

Поддержка системы

Данная система может быть запущена на большинстве существующих серверах. Поддержка браузерами системы визуального редактирования также велика.

Пользователи

В CMS имеется поддержка базы пользователей.

Информация о пользователях представляется в удобном виде (таблица).

Распределение прав. Статусы

Права

Категории прав

Для удобного управления правами в данной системе предусмотрены категории прав.

С ними возможны следующие операции:

  • Создание новой категории
  • Изменение уже существующей
  • Удаление категории (права этой категории теперь не будут иметь какой-либо категории)
Управление правами

В CMS Leonid статусам пользователей назначаются права.

С правами возможно производить следующие операции

Управление статусами

В данной системе пользователи имеют статус. Статус определяет права группы пользователей.

Со статусами можно производить следующие операции:

  • Создавать статус
  • Изменять название статуса
  • Назначать права статусу
  • Удалять статус
  • Изменять порядок статусов:
    • Повышение
    • Понижение

Управление пользователями

Имеется возможность проводить следующие операции с пользователями:

  • Вход от имени другого пользователя
  • Назначение продвинутых пользователей
  • Создавать пользователей
  • Сменять пароль пользователям
  • Удалять пользователей
  • Сменять статусы пользователей
    • Понижение пользователя
    • Повышение пользователя

Отслеживание

В системе имеется возможность отслеживать действия пользователей.

Для реализации отслеживания используются следующие средства:

  • Ведётся лог всех действий пользователя
  • Имеется возможность просмотреть текущее (или последнее) действие

Страницы

В данной системе имеется поддержка страниц.

Страницы основаны на выбранном для неё шаблоне, полях шаблона (таких как заголовок страницы и т.п.) и частях. Отслеживать действия со страницами возможно с помощью специального лога.

Шаблоны

В этой системе для создания удобного страниц используются шаблоны.

Управление шаблонами

С шаблонами возможно производить следующие операции:

  • Создать новый шаблон
  • Изменять шаблон
  • Изменять стили шаблона (изменение стиля шаблона происходит в удобном интерфейсе)
  • Копировать шаблон
  • Перемещать шаблон (менять имя)
  • Удалять шаблон

Назначение шаблонов

В этой CMS шаблон можно выбирать как:

  • Шаблон CMS (не рекомендуется использовать шаблон не предназначенный для этого)
  • Шаблон по умолчанию (этот шаблон будет использован страницами, если при их создании был выбран соответствующий пункт меню поля "Шаблон" или при ошибке в обращении к выбранному шаблону)

Пакеты шаблонов

В системе CMS Leonid для удобства пользователя имеется возможность использовать пакеты шаблонов. С ними возможно производить следующие действия:

  • Установка пакета шаблонов
  • Откат изменений пакета шаблонов
  • Создание нового пакета шаблонов
  • Удаление пакета шаблонов
  • Копирование пакета шаблонов
  • Перемещение пакета шаблонов (смена имени)

Части шаблонов

В шаблонах имеются элементы, которые необходимо изменять редко (например, название сайта). Чтобы не пришлось изменять шаблон, который имеет HTML синтаксис, имеется возможность изменить часть шаблона, которая отвечает за это.

С частями возможно производить следующие операции:

  • Копирование
  • Переименование
  • Создание
  • Изменение
  • Удаление
  • Выбор как панель управления HTML при создании страниц.
  • Выбор как панель навигации CMS (для плагинов)
Пакеты частей

Для удобного манипулирования частями имеются пакеты частей

  • Установка пакета частей
  • Откат изменений пакета частей
  • Создание новаго пакета частей
  • Удаление пакета частей
  • Копирование пакета частей
  • Перемещение пакета частей (смена имени)

Управление

Со страницами возможно производить следующие действия:

  • Создание
  • Изменение
  • Удаление страницы
  • Удаление директории

Категории

Категории позволяют получить удобный доступ к страницам.

С категориями возможно производить следующие действия:

  • Создание
  • Изменение
  • Удаление

Управление HTML тегами

Страницы основаны на HTML и, поэтому, необходимо разрешения HTML тегов.

Со списком HTML тегов возможно производить следующие операции:

  • Добавление тегов
  • Добавление атрибутов
  • Удаление тегов
  • Удаление атрибутов

Конфигурация и управление системой

Данную систему пользователь может настраивать и обновлять. Для этого присутствуют многочисленные модули CMS.

Модульность

Система основывается на модулях. Присутствует основной модуль (класс CMS), модули пользовательского интерфейса (страницы CMS), модули обработки (Действия) и вспомогательные модули (другие классы). Но эту систему можно дополнять с помощью пользовательских модулей (плагинов и скриптовых модулей).

Скриптовые модули

Скриптовые модули - это модули PHP, CSS, JavaScript, которые подключаются на страницу. Например, часы.

С ними возможно производить следующие действия:

  • Установка
  • Откат изменений
  • Создание
  • Изменение
  • Переименование
  • Удаление
  • Обновление

Плагины

Плагин - универсальный модуль системы. Они могут производить любые действия.

С ними можно производить следующие действия:

  • Установка
  • Полная деинсталляция
  • Деинсталляция с сохранением настроек
  • Создание
  • Удаление
  • Обновление

Конфигурация

Система автоматически обновляется.

Настройки

В данной системе возможно производить настройку системных файлов таких как INI и HTACCESS

Архивация и восстановление

Файлы данной системы и сайта возможно архивировать.

С архивами производить следующие действия:

  • Создание
  • Создание низкого уровня
  • Проверка на файловые ошибки
  • Разархивирование и применение
Настройка баз данных и путей

В данной системе возможно настраивать временную зону, принудительно выставлять версию, менять затравку безопасности, очищать защитный счётчик, исправлять список страниц и многое другое.

Настройки баз расширений

В данной системе возможно изменять базы данных расширений. Изменяются базы типов и типов MIME.

Настройки путей

Пользователь может настроить служебные пути. А именно:

  • Путь магазина
  • Путь помощи
  • Путь обновлений
Переменные окружения

В данной системе имеется возможность просматривать переменные окружения. В них входят системные константы, просмотр массивов ($_SERVER, $_ENV), просмотр информации о php (phpinfo()).

Сканер функциональности PHP

В сканер функциональности PHP входит:

  • Скорость хеширования
  • Скорость действий CMS
  • Выделенная память
  • Постепенный вывод
  • Все функции
  • Все классы
  • Все расширения
Управление ошибками и безопасностью

В данной системе имеется возможность управлять ошибками и безопасностью. Пользователи, имеющие права могут отключать стандартные средства защиты (затравку или защитный счётчик). В системе ведётся лог ошибок. Ему можно задать максимальный размер.

Модуль error

В системе CMS Leonid имеется условно называемый модуль error. Это автоматическое средство сбора информации об ошибках и исправления некоторых из них. Исправление ошибок - это разрешение на перезапись файла дампом памяти, который не должен содержать ошибки.

Модуль protector

В системе CMS Leonid имеется условно называемый модуль protector. Это средство автоматического уменьшения нагрузки на сервер в условиях угрозы.

Файлы

Данная система основана на файлах, которые можно подгружать (в галерею или как системный объект), скачивать, управлять с помощью плагина file. Имеется возможность настраивать этот плагин (выделение цветом и настройка расширений).

Настройка интерфейса. Темы

Интерфейс пользователя может быть настроен в соответствии с его пожеланиями и предпочтениями. Возможно выбирать тему для системы и фон сайта. Темы для удобства объединены в пакеты.

Темы возможно настраивать как тему CMS или как фон сайта.

Темы можно подгружать, удалять (удаление одной темы, массовое удаление тем, удаление дублей).

Возможно устанавливать пакеты тем, производить откат изменений, создавать, переименовывать, копировать, удалять.
